# Canonical Strategic Brief (`context.md`)

## 1. Technical Event Context

- **Event Description**: Reference implementation demonstrating how to structure a multi-platform content packet in a Git-first repository.
- **Why It Matters**: Technical creators often scatter drafts across informal notes apps. This packet demonstrates how canonical engineering context and channel adaptations co-exist in version control.

## 2. Target Audience Segments

- **Primary Audience**: Engineering leaders, platform architects, and developer advocates on LinkedIn and technical blogs.
- **Secondary Audience**: DevOps practitioners and open-source creators on X and Facebook.

## 3. Core Messaging Thesis

- **Primary Thesis**: Content architecture should be structured like software engineering systems: single canonical source of truth with specialized output adapters.
- **Supporting Arguments**:
  - Argument 1: Git version control prevents specification drift and provides clear ownership history.
  - Argument 2: Isolating channel adaptations inside `channels/` prevents root folder clutter.
  - Argument 3: Machine-readable JSON Schema files enforce consistent metadata across automated pipelines.

## 4. Tone and Editorial Constraints

- **Tone**: Direct, instructional, technical, and structured.
- **Constraints**: No em dashes; adhere strictly to platform formatting rules.
