# Technical Blog Platform Guidelines

## 1. Target Audience

Engineers and architects seeking deep, rigorous, end-to-end technical explanations, architecture patterns, and reproducible implementations.

## 2. Structural Standards

- **Title**: Clear, keyword-rich, descriptive technical headline.
- **Hierarchy**: Use clear markdown headings (`##` and `###`), structured comparison tables, and Mermaid architecture diagrams where applicable.
- **Code Snippets**: Include concrete, formatted code examples, CLI commands, or YAML configuration blocks.
- **Length**: Long-form article (800 to 2500 words).

## 3. SEO and Syndication Standards

- Every blog adaptation must include a descriptive hook in front matter (`content_spec.hook`).
- If syndicated from an external publication, populate `distribution.syndication_canonical_url`.
- Never use em dash characters.
